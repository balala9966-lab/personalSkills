# Composition Patterns

## Use These Structures

- Threshold: the character crosses a boundary, membrane, door, checkpoint, or weather line.
- Split path: one input branches into different consequences.
- Loop: inputs return as feedback, debt, learning, or compounding effects.
- Compression: many messy pieces are squeezed into one usable form.
- Translation: one format becomes another through an odd machine or ritual.
- Balance: competing forces are held in tension.
- Excavation: hidden causes are dug out from under visible symptoms.
- Relay: one role hands off to another through a fragile interface.
- Repair: the character fixes a leak, crack, knot, or broken connector.
- Mismatch: two systems try to interlock but have incompatible shapes.

## Invent The Metaphor

Start from the article's actual claim:

1. Identify the transformation: before -> after, input -> output, noise -> signal, promise -> cost.
2. Choose a physical action that embodies it.
3. Add one strange but disciplined object relationship.
4. Add only enough labels to make the metaphor readable.

## Anti-Repetition

Do not repeatedly use the same visual moves across images:

- Conveyor belts.
- Pulling a single rope or cable.
- Fish as reusable content.
- Stamp/toolbox productivity metaphors.
- "Pitfall path" diagrams.
- Generic funnels.
- Puzzle pieces.
- Ladders and mountains.
- Brain with gears.
- Lightbulbs as "ideas".

Reuse a composition only when the user explicitly asks to mimic a reference.

## Keep It Editorial

The image should feel like a body illustration inserted between paragraphs. It should not feel like:

- A slide cover.
- A process manual.
- A SaaS landing-page hero.
- A technical architecture diagram.
- A children's book spread.
