# Prompt Template

Use this template for one image at a time:

```text
Create a 16:9 horizontal editorial body illustration for an article.

Style: pure white background, minimal hand-drawn product-sketch feel, thin black organic line work with slight wobble, lots of whitespace, restrained, weird, clean and crisp, like an AI/product/design/dev-tools expert casually sketching on a blank sheet. Subject fills about 40%-60% of the frame, with at least 35% empty space and one quiet area.

Core idea: [ONE-SENTENCE IDEA]

Scene: [ODD BUT PRECISE METAPHORICAL SCENE]

Character: The Tinkerer (or the user-supplied IP) is the active subject, doing [CORE ACTION]. If using the default Tinkerer, preserve these traits: a clean round head drawn in thin black line, two small upright tufts on top of the head (the silhouette signature, never omit), two small dot eyes, a tiny neutral mouth, a small compact body in a plain white coverall with no logo / no patch / no readable text / no chest emblem, simple mitten-like hands actively doing the action, plain rounded shoes. Body is black line work on white only; no colored fill on the figure. Translate it into the hand-drawn product-sketch style; do not render it as glossy 3D, mascot poster, or polished vector unless explicitly requested. The character is not a decoration or narrator.

Composition: [STRUCTURE TYPE], one main scene, readable at article width. Let the structure come through naturally; do not write the structure type in the image.

Color: black for main line work and primary text; red for key annotations / problems / outcomes; orange for main flow / arrows / movement; blue only for optional second-layer notes or AI / system-state hints. Keep color sparse.

Handwritten labels: [5-8 SHORT LABELS IN LANGUAGE], each ideally 2-6 words.

Avoid: top-left title, big headline, dense paragraphs, flowchart boxes, business infographic style, formal flowchart, lecture slide, instruction manual, childish mascot style, polished flat vector, techy UI, real app screenshots, dark/colored background, beige/warm gray/paper texture, gradients, shadows, noise, any company logo, any brand wordmark, any readable text on the character's clothing.
```

If revising an existing image, preserve the strongest parts and ask for concrete edits only when the user's request is ambiguous.
