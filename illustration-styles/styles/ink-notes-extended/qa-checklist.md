# QA Checklist (ink-notes extended)

Before delivering or accepting a generated `ink-notes` editorial illustration, verify every item. If two or more fail, regenerate — don't explain the problem away.

## Must Pass

- [ ] 16:9 horizontal composition
- [ ] Clean pure white background (no beige, warm gray, paper texture, gradients, shadows, noise)
- [ ] Thin black hand-drawn line work dominates, with slight organic wobble (not vector-smooth)
- [ ] Red/orange/blue accents are sparse — less is better
- [ ] Subject fills roughly 40-60% of the frame
- [ ] At least 35% of the image is empty whitespace, including one whole quiet area
- [ ] The character (Tinkerer or user IP) performs the central action, not decoration
- [ ] The concept maps to the article's specific idea, not a vague mood
- [ ] One clear structure, action, state, or metaphor per image
- [ ] Labels in the requested language (default English)
- [ ] Labels are short and legible: 5-8 spots max, ideally 2-6 words each
- [ ] No structure-type label visible in the image (no "Workflow", "Architecture", etc.)
- [ ] No top-left diagram title
- [ ] No company logo, brand wordmark, or readable text on character clothing

## Must NOT Appear

- [ ] PPT flowchart / consulting infographic
- [ ] Course slide / instruction manual aesthetic
- [ ] Glossy vector / polished flat illustration / 3D render / stock look
- [ ] Techy UI / real app screenshot
- [ ] Childish/cute treatment (unless user explicitly asks)
- [ ] Busy background / colored background
- [ ] Dense paragraphs of text in the image
- [ ] Multiple unrelated concepts crammed into one frame

## Character-Specific Checks

For The Tinkerer:
- [ ] Round head with two upright tufts present
- [ ] No colored fill on body (black line only)
- [ ] No logo/text on clothing
- [ ] Face is minimal (dots + line, not detailed)

For user-supplied IP:
- [ ] Silhouette matches user's reference/description
- [ ] IP is driving the action (not watching)
- [ ] No Tinkerer mixed in unless explicitly asked
- [ ] If IP has a canonical color, only that color appears on the character

## Composition Pattern Check

- [ ] The chosen pattern (Threshold/Loop/Balance/etc.) is visually clear without needing a label
- [ ] The metaphor is specific to this article, not a generic visual cliché
- [ ] No items from the anti-repetition list appear (see composition-patterns.md)

## When To Regenerate vs Iterate

| Situation | Action |
|-----------|--------|
| 2+ checklist items fail | Regenerate from scratch |
| 1 item fails but overall strong | Request a specific edit |
| Character is decorative | Regenerate — the core rule is broken |
| Image is too busy | Regenerate — whitespace is structural |
| Wrong structure type | Regenerate with corrected pattern |
| Labels misspelled / wrong language | Request text edit only |
